Base URL: **`http://localhost:8585/api`** ┬╖ Most catalog endpoints are auto-generated by
**Spring Data REST** and return HAL JSON (`application/vnd.hal+json`).

| Group | Method | Path | Auth |
|---|---|---|---|
| [Catalog](#catalog) | GET | `/products` | public |
| | GET | `/products/{id}` | public |
| | GET | `/products/search/findByCategoryId` | public |
| | GET | `/products/search/findByNameContaining` | public |
| | GET | `/product-category` | public |
| [Reference data](#reference-data) | GET | `/countries` | public |
| | GET | `/states/search/findByCountryCode` | public |
| [Checkout](#checkout) | POST | `/checkout/purchase` | public |
| | POST | `/checkout/payment-intent` | public |
| [Orders](#orders-secured) | GET | `/orders/search/findByCustomerEmailOrderByDateCreatedDesc` | ЁЯФТ JWT |

## Pagination envelope

Every collection endpoint returns the Spring Data REST HAL shape тАФ this is exactly what the Angular
services unwrap:

```jsonc
{
  "_embedded": { "products": [ /* ... */ ] },
  "_links":    { "self": { "href": "..." } },
  "page": { "size": 12, "totalElements": 20, "totalPages": 2, "number": 0 }
}
```

## Catalog

### List products (paginated)
```bash
curl "http://localhost:8585/api/products?page=0&size=12"
```

### Products in a category
```bash
curl "http://localhost:8585/api/products/search/findByCategoryId?id=1&page=0&size=12"
```

### Keyword search
```bash
curl "http://localhost:8585/api/products/search/findByNameContaining?name=Action"
```

### Categories
```bash
curl "http://localhost:8585/api/product-category"
# -> _embedded.productCategory: [{ id, categoryName }]
```

## Checkout

### Place an order
```bash
curl -X POST "http://localhost:8585/api/checkout/purchase" \
  -H "Content-Type: application/json" \
  -d '{
    "customer":       { "firstName":"Ada", "lastName":"Lovelace", "email":"ada@example.com" },
    "shippingAddress":{ "street":"1 Test St","city":"Testville","state":"California","country":"United States","zipCode":"90210" },
    "billingAddress": { "street":"1 Test St","city":"Testville","state":"California","country":"United States","zipCode":"90210" },
    "order":          { "totalQuantity":2, "totalPrice":21.98, "status":"Received" },
    "orderItems":     [ { "imageUrl":"...", "quantity":2, "unitPrice":10.99, "productId":1 } ]
  }'
# -> { "orderTrackingNumber": "a8a78b09-...." }
```

The server generates the tracking number and cascade-persists `customer тЖТ order тЖТ orderItems тЖТ
shipping/billing addresses`.

### Create a Stripe PaymentIntent
```bash
curl -X POST "http://localhost:8585/api/checkout/payment-intent" \
  -H "Content-Type: application/json" \
  -d '{ "amount": 2198, "currency": "USD", "receiptEmail": "ada@example.com" }'
# -> the Stripe PaymentIntent JSON (contains client_secret)
```
Requires `STRIPE_SECRET_KEY` тАФ see [STRIPE.md](STRIPE.md). `amount` is in the smallest currency unit (cents).

## Orders (secured)

Read-only and protected by JWT **once an Okta issuer is configured**
(`spring.security.oauth2.resourceserver.jwt.issuer-uri`). The Angular `authInterceptor` attaches the
bearer token automatically.

```bash
curl "http://localhost:8585/api/orders/search/findByCustomerEmailOrderByDateCreatedDesc?email=ada@example.com" \
  -H "Authorization: Bearer <token>"
```

## Conventions

- **CORS:** allowed origins `http://localhost:4200` and `http://localhost:4250`.
- **Writes disabled:** `POST/PUT/PATCH/DELETE` are turned off on the auto-exposed domain types
  (`MyDataRestConfig`) тАФ the catalog is read-only; orders are created only via `/checkout/purchase`.
- **IDs exposed:** entity ids are included in responses (`exposeIdsFor`).
